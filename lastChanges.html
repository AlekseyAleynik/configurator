Версия 1.1.6
  • Изменён функционал логгирования.
  • Добавлена возможность просмотра изменений в версиях программы.

Версия 1.1.7
  • Исправлена работа механизма отслеживания помех связи ОУ с БРШС.
  • Исправлен алгоритм привязки интерфейсных устройств.
  • Изменено значение по умолчание "комбобокса", определяющего номер канала для связывания устройств
    при выставленной галочке в "чекбоксе", определяющем тип устройства.
  • Устранено "выпадение" лишних сообщений при потери связи с БРШС.

Версия 1.1.8
  • Сообщения о занимании канала логгируется теперь только при связывании устройства.
    В текстовом поле на вкладке "Монитор" это сообщение теперь не дублируется"

Версия 1.1.9
  • Исправлена ошибка отражения тревоги для ОУ 4 класса Цельций-РК
  • Добавлена поддержка ОУ 4 класса Цельсий-РК новой версии (4115)

Версия 1.1.10
  • Добавлено отображение сообщений о помехах в текстовое поле во вкладке "Монитор"
  • Исправлен алгоритм регистрирования уровня сигнала связи с ОУ (теперь время не ограничивается одними сутками).

Версия 1.1.11
  • Изменена работа программы. Начиная с этой версии, в таблицы на вкладке "Состав", после связывания ОУ
    выделяется первая слева ячейка строки с информацией о последнем присоединённом устройстве.

Версия 1.1.12
  • Исправлена фильтрация записей в "комбобоксе" выбора подключаемых устройств в зависимости от
    открытой вкладки в разделе "Состав"
  • Добавлен дополнительный опрос состояния подключенных ОУ (команда 0x01) через промежутки времени
    равные периоду свазывания отдельных ОУ(Тестируется).

Версия 1.1.13
  • Исправлена отражение смены помеховой обстановки в эфире для БРШС в логе и текстовом окне на вкладке "Монитор"
  • Исправлены ошибки появляющиеся при формировании пакетов запросов состояния OУ 2-го класса (команда 0x21)

Версия 1.1.14
  • Добавлена поодержка одноканальных аналоговых извещателей Цельсий-РК
  • Исправлен алгоритм обработки ошибок при запросах 0x12 и 0х13(работа с аналоговые характеристики)
    для аналоговых датчиков

Версия 1.1.15
  • В соответствии с изменением в протоколе изменен алгоритм парсинга пакетов ответа от БРШС по команде 0x69
    (В протоколе было изменено расположение битов помехи)
  • Исправлена ошибка отображения изменений настроек перед нажатием пользователем конпки "Установить"
    Ранее, измененные, но неустановленные (не переданные ОУ) значения приводились в исходное состояние,
    как только проходила команда 0x01 (0x21, 0x41, 0x81). Это происходило примерно 1 раз в 30 сек

Версия 1.1.16
  • Добавлена блокировка редактирования временных параметров режимов работы аналоговых ОУ "Трубач-РК"
    для первых 7 режимов
  • Окно "Дерево устройств" теперь изменяет свой размер с изменением размера окна приложения
  • Изменено именование колонок в таблицах на вкладках "Состав" и "Мониторинг"
  • У подключенных ОУ, с которыми потерян сигнал, теперь длительность периода соединения подсвечивается серым цветом
    (при восстановлении соединения цвет надписи восстанавливается в чёрный)
  • Убрана за ненадобностю чекбокс "Log" во вкладке "Мониторинг"
  • Для закладок "Все" и "Дерево устройств раздела "Состав" теперь блокируются панель с инструментами для связывания
  • Для закладки "Интерфейсные" Добавлена фильтрация доступных для связывания интерфейсных ОУ
  • Добавлено логгирование потери соединения с конфигуратора с БРШС и автоматическое восстановление
    в случае, если соединения

Версия 1.1.17
  • Установление настройки по умолчанию в операционной системе пути хранения лог-файла
  • Добавлена поддержка одноканального режима работы двухканальных устройств
  • Исправлено отображение двухканальных ОУ на вкладке "Все" в разделе "Состав"
  • Сообщения об ошибках теперь отображаются только в рассширенном режиме (когда открыта вкладка "Команды")

Версия 1.1.18
  • Добалена поддержка МК-РК ипс.2 и СТЗ-РК исп.1
  • Кнопки "Установить" и "Отмена" отображаются теперь всегда.
  • Добавлено отображение значения первого байта на вкладке мониторинг в таблице в случае Тревоги.
  • Добавлено отображение неисправности на вкладке мониторинг в таблице в случае наличия в пакете ответа единицы
    в бите неисправности.

Версия 1.1.19
  • Добавлено ограничение выбора значений порогов для ОУ Цельсий-РК
  • Реализован механизм возвращения старого значения параметров в случае получения ошибки при выпонении команды
Версия 1.1.23
  • Исправлены ошибки в работе на вкладке дерево устройств, появляющиеся при работе с устройствами ОУ 4-го класса.
  • Добавлено отображение номера подключенного порта в шапке окна приложения.
  • Теперь после загрузки прошивки автоматически назначается скорость соединения, которая была установлена при
    запуске приложения.
  • Реализована принудительное назначение пути лог-файла.
  • Внесены изменения в интерфейс работы с ОУ 2 класса "Трубач-РК".
Версия 1.1.24
  • Исправлена ошибка привязки устройств, связанная с обработкой исключительной ситуации аварийной остановки.
  • Изменен алгоритм запросов значений установленных параметров работы ОУ. Теперь время между опросами соотетствует
    времени проходимому между сеансами связи, а не времени периода контроля, как было ранее.
  • Теперь, вне зависимости от состояния чекбокса "память тревоги/потери" (вкладка Мониторинг)
    сообщения о потерях и тревогах сообщения попадают в текстовое поле с кратким логом слева на вкладке Мониторинг.
  • Добавлена фильтрация аналоговых устройств при отображении журнала.
Версия 1.1.30
  • Исправлена ошибка управления звуковой индикации (ранее были перепутаны поля в таблице на вкладке состав).
  • Добавлены новые сообщения в систему логгирования и исправлены ошибки в этой системе.
  • Добавлена обработка кодов ошибок в байте данных для аналоговых датчиков.
Версия 1.1.38
  • Номер порта в шапке приложения теперь изменяется с изменением в порта по выбору во вкладке подключить.
  • Добавлена фильтрация сообщений по чекбоксу во вкладке мониторинг.
  • В колонках "Уровень сигнала" теперь при подсчете среднего количества не учитываются неактивные каналы.
